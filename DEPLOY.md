# Deploy SK Academy — Vercel + Render + MongoDB Atlas

## Overview

| Service | Host | What runs |
|---------|------|-----------|
| Frontend | **Vercel** | Next.js (`/`) |
| Backend | **Render** | Express API (`server/`) |
| Database | **MongoDB Atlas** | Already configured |

Traffic flow: browser → Vercel → rewrites `/api/*` and `/uploads/*` → Render → Atlas.

---

## 1. MongoDB Atlas

1. Open [MongoDB Atlas](https://cloud.mongodb.com).
2. **Network Access** → allow `0.0.0.0/0` (or Render’s outbound IPs if you restrict later).
3. Copy your connection string (encode `@` in passwords as `%40`).
4. Keep it for Render env vars.

---

## 2. Push code to GitHub

```bash
git add .
git commit -m "Prepare Vercel + Render deployment"
git push origin main
```

---

## 3. Deploy backend on Render

### Option A — Blueprint (recommended)

1. Go to [render.com/dashboard](https://dashboard.render.com) → **New** → **Blueprint**.
2. Connect your GitHub repo.
3. Render reads `render.yaml` and creates **sk-academy-api**.
4. Set these env vars when prompted (or in the service **Environment** tab):

| Variable | Example |
|----------|---------|
| `MONGODB_URI` | `mongodb+srv://user:pass@cluster.mongodb.net/sk-academy` |
| `CLIENT_URL` | `https://your-app.vercel.app` (set after Vercel deploy) |
| `ADMIN_EMAIL` | `admin@skacademy.edu` |
| `ADMIN_PASSWORD` | Strong password for production |

`JWT_SECRET` is auto-generated by the blueprint.

### Option B — Manual web service

1. **New** → **Web Service** → connect repo.
2. **Root Directory:** `server`
3. **Build Command:** `npm install && npm run build`
4. **Start Command:** `npm start`
5. **Health Check Path:** `/health`
6. Add the same env vars as above.

### After deploy

1. Copy your Render URL, e.g. `https://sk-academy-api.onrender.com`.
2. Test: open `https://sk-academy-api.onrender.com/health` → should return `{"status":"ok"}`.

### Seed the database (once)

In Render → your service → **Shell**:

```bash
npm run seed
```

Or locally (with Atlas URI in `server/.env`):

```bash
npm run seed
```

---

## 4. Deploy frontend on Vercel

1. Go to [vercel.com/new](https://vercel.com/new) → import your GitHub repo.
2. **Framework:** Next.js (auto-detected).
3. **Root Directory:** `.` (repo root, not `server`).
4. **Environment Variables:**

| Name | Value |
|------|-------|
| `API_URL` | `https://sk-academy-api.onrender.com` |

No trailing slash. Apply to **Production**, **Preview**, and **Development**.

5. Click **Deploy**.

---

## 5. Link frontend and backend

1. Copy your Vercel URL, e.g. `https://sk-academy.vercel.app`.
2. In **Render** → **Environment** → set:

```
CLIENT_URL=https://sk-academy.vercel.app
```

3. **Save** and let Render redeploy (or manual redeploy).

## Notes

### Free Render tier
- Service **sleeps** after ~15 min idle; first request may take 30–60s.
- Upgrade to a paid plan for always-on production use.

### Image uploads
- Uploads are stored on Render’s local disk and **may be lost on redeploy**.
- For production, migrate to Cloudinary or S3 later.

### Custom domain
- Point your domain to Vercel.
- Update `CLIENT_URL` on Render to `https://yourdomain.com`.
- Keep `API_URL` on Vercel as the Render URL (rewrites handle routing).

### Redeploy checklist
1. Push to `main` → Vercel auto-deploys.
2. Render auto-deploys if connected to the same branch.
3. Re-run seed only on a fresh database.
